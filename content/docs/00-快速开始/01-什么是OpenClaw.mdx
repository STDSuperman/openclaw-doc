# 什么是 OpenClaw？

> **TL;DR**: OpenClaw 是一个可运行在自己设备上的**个人 AI 助手**，支持 WhatsApp、Telegram、Discord、iMessage 等多平台消息通道。

---

## 核心特性

### 🤖️ 多平台支持

OpenClaw 连接到你日常使用的消息平台：

| 平台 | 功能 | 状态 |
|------|------|------|
| **WhatsApp** | 二维码登录、群聊、图片/视频/语音 | ✅ 核心 |
| **Telegram** | Bot 模式、私聊、群组、机器人命令 | ✅ 核心 |
| **Discord** | 服务器模式、频道、私聊、表情 | ✅ 核心 |
| **Slack** | 工作区集成、频道、私聊 | ✅ 核心 |
| **Google Chat** | Google Workspace 集成 | ✅ 核心 |
| **Signal** | 端到端加密消息 | ✅ 核心 |
| **iMessage** | macOS 本地集成 | ✅ 核心 |
| **Microsoft Teams** | 工作区聊天 | 🔌 插件 |
| **Matrix** | 去中心化协议 | 🔌 插件 |

### 🎯 核心能力

#### 1. 智能对话

- ✅ **上下文理解**：记住对话历史
- ✅ **工具调用**：执行命令、运行脚本、浏览网页
- ✅ **多智能体**：不同消息使用不同的 AI 配置
- ✅ **流式响应**：实时打字效果
- ✅ **多模态输入**：文本、图片、语音、文件

#### 2. 自动化任务

- ✅ **定时任务**：Cron 定时触发
- ✅ **Webhook**：外部事件触发
- ✅ **Gmail 集成**：邮件监听和回复
- ✅ **自定义技能**：扩展 AI 能力

#### 3. 设备集成

- ✅ **macOS 应用**：菜单栏控制、Voice Wake
- ✅ **iOS 节点**：Canvas、相机、录音
- ✅ **Android 节点**：Canvas、相机、屏幕录制
- ✅ **远程访问**：SSH 隧道、Tailscale

## 工作原理

### 架构概览

```
┌─────────────────────────────────────┐
│        用户消息平台              │
│  (WhatsApp/Telegram/Discord)      │
└────────────┬────────────────────────┘
             │
             ▼
┌─────────────────────────────────────┐
│         Gateway（网关）            │
│    ws://127.0.0.1:18789          │
│  - 消息通道管理                │
│  - 会话管理                      │
│  - 工具调用                      │
│  - AI 智能体路由                │
└────────────┬────────────────────────┘
             │
             ▼
┌─────────────────────────────────────┐
│        Pi Agent（AI 智能体）        │
│  - 上下文处理                  │
│  - 工具执行                      │
│  - 响应生成                      │
└─────────────────────────────────────┘
```

### 数据流向

1. **用户** → 在 WhatsApp/Telegram/Discord 发送消息
2. **Gateway（网关）** → 接收消息，管理会话，调用 Pi Agent
3. **Pi Agent** → 理解请求，调用工具（浏览器、文件读写、脚本执行）
4. **返回结果** → Gateway → 消息平台

## 与其他 AI 助手的区别

| 对比项 | OpenClaw | ChatGPT 网页版 | 其他托管助手 |
|---------|---------|-------------|-------------|
| **部署方式** | 自己设备运行 | 云端托管 | 云端托管 |
| **数据隐私** | 完全本地 | 需要信任服务商 | 需要信任服务商 |
| **消息通道** | 集成现有应用 | 只能网页对话 | 受限的平台 |
| **可定制性** | 高度可定制 | 无法定制 | 低定制性 |
| **成本** | 仅 API 费用 | 可能包含订阅费 | 订阅 + API 费用 |
| **控制权** | 完全控制 | 无法控制 | 无法控制 |

**OpenClaw 的核心优势**：
- 🔒 数据完全本地存储，完全掌控
- 🚀 无需打开浏览器即可使用
- 🎨 支持你熟悉的消息应用
- 🔧 高度可定制（工具、技能、智能体）
- 💰 成本可控，仅支付 AI API 费用

## 使用场景

### 场景 1：个人知识库助手

**需求**：快速查询个人笔记、文档、项目信息

**配置**：
- 连接 WhatsApp/Telegram
- 设置工作空间目录（`~/.openclaw/workspace`）
- 添加自定义技能（代码搜索、文档查询）

**效果**：
- 💬 直接在消息中询问问题

- 🔄 支持知识库更新和同步

### 场景 2：工作自动化

**需求**：自动化日常任务、定时提醒

**配置**：
- 设置 Cron 定时任务

- 添加脚本执行工具

**效果**：
- ⏰ 定时发送提醒、报告

## 技术架构关键词

### Gateway（网关）
OpenClaw 的核心控制平面，管理所有组件：

- **WebSocket API**：为所有客户端提供统一接口
- **会话管理**：维护对话上下文和历史
- **工具注册表**：暴露系统工具（浏览器、文件、命令）
- **消息通道适配器**：支持 WhatsApp、Telegram、Discord 等

### Pi Agent
OpenClaw 默认的 AI 智能体：

- **内置工具**：浏览器控制、文件读写、脚本执行
- **流式响应**：实时生成和发送回复
- **多智能体支持**：每个会话可以有不同的配置和工具权限
- **会话修剪**：自动压缩旧对话，节省 token

### Nodes（节点）
设备节点系统，支持本地设备集成：

- **设备配对**：安全的设备发现和连接
- **能力声明**：设备声明支持的功能（Canvas、相机、麦克风）
- **远程调用**：通过 Gateway 协议调用设备功能
