# 术语表

> **TL;DR**: OpenClaw 的核心术语和概念词汇表，帮助理解文档中的技术词汇。

---

## A

### Agent
- **定义**：OpenClaw 使用的 AI 智能体，基于 Pi 架构
- **特点**：内置工具调用、流式响应、多会话管理
- **同义词**：AI Assistant、AI Bot、智能体、助手

---

## B

### Binding
- **定义**：将消息路由规则绑定到特定的智能体或通道
- **用途**：控制消息如何被路由到哪个智能体或通道
- **示例**：将 `+86138000000` 的消息路由到 `agent:custom:workspace`
- **同义词**：路由规则、绑定配置、消息分发

---

## C

### Channel（通道）
- **定义**：OpenClaw 支持的消息平台（WhatsApp、Telegram、Discord 等）
- **同义词**：消息平台、通讯渠道、社交平台
- **类型**：第三方平台、内置平台（iMessage）、插件平台（Mattermost）

---

## D

### Daemon（守护进程）
- **定义**：在后台持续运行的系统服务
- **同义词**：后台服务、系统服务、服务进程、长期运行

---

## E

### Embedding（嵌入）
- **定义**：将文本转换为数值向量，用于语义搜索和 RAG
- **用途**：实现语义搜索和上下文相似性计算
- **模型**：OpenAI text-embedding-3-small、Anthropic Claude 等

---

## F

### Frontmatter
- **定义**：Markdown 文件顶部的 YAML 元数据块
- **用途**：为文档添加元数据（标题、描述、标签、作者等）
- **示例**：
  ```yaml
  ---
  title: "配置文件说明"
  tags: ["配置", "JSON5"]
  author: "OpenClaw Team"
  ```

---

## G

### Gateway
- **定义**：OpenClaw 的核心控制平面，管理所有组件
- **同义词**：网关、控制平面、消息中心、统一入口

---

## H

### Health Check（健康检查）
- **定义**：检查 Gateway 和所有连接的状态
- **用途**：快速判断系统是否正常运行
- **命令**：`openclaw health`
- **返回**：运行时间、通道状态、模型状态、内存使用等

---

## I

### Identity（身份）
- **定义**：AI 或智能体的身份和配置
- **文件**：`~/.openclaw/agents/<agentId>/agent/IDENTITY.md`
- **配置项**：名字、主题、语气、格式偏好

---

## J

### JSON5
- **定义**：支持注释和尾随逗号的 JSON 格式
- **用途**：配置文件格式
- **示例**：
  ```json5
  {
    "gateway": {
      "port": 18789,
      "bind": "loopback"
    }
  }
  ```

---

## K

### Key（密钥）
- **定义**：用于加密认证或 API 访问的密钥
- **类型**：
  - `sk-xxx`: API 密钥（如 Anthropic）
  - `token-xxx`: 认证令牌
  - `xxx.pem`: TLS 证书
  - `xxx`: 私钥

---

## L

### LLM（Large Language Model）
- **定义**：大型语言模型（如 GPT-4o、Claude 3.5 等）
- **同义词**：模型、大语言模型、AI 模型、LLM

---

## M

### Message
- **定义**：在消息通道中发送的文本、图片、文件等内容
- **同义词**：消息、报文、多媒体、附件

---

## N

### Node.js
- **定义**：OpenClaw 依赖的 JavaScript 运行时环境
- **要求**：Node.js >= 22
- **推荐**：Node.js 22 LTS 或更高

---

## O

### OAuth
- **定义**：开放授权标准，用于安全地连接第三方服务（如 Anthropic Claude）
- **流程**：用户授权 → 重定向回 Gateway → 交换授权码
- **同义词**：OAuth 2.0、授权、第三方认证

---

## P

### Pi
- **定义**：OpenClaw 内置的默认 AI 智能体
- **特点**：基于原始项目架构设计
- **作者**：Mario Zechner
- **GitHub**: https://github.com/badlogic/pi-mono

---

## Q

### Queue（队列）
- **定义**：消息队列系统，管理待处理的消息
- **功能**：优先级控制、消息排序、重试机制
- **配置**：`routing.queue`

---

## R

### Request
- **定义**：客户端通过 WebSocket 发送到 Gateway 的请求
- **类型**：`req`（请求）、`res`（响应）、`event`（事件）

---

## RPC
- **定义**：Remote Procedure Call（远程过程调用）
- **用途**：客户端通过 WebSocket 调用 Gateway 的方法
- **示例**：`config.get`、`agent`、`send` 等

---

## S

### Session（会话）
- **定义**：OpenClaw 的对话上下文管理单元
- **功能**：维护对话历史、管理会话状态、压缩旧对话
- **Key 格式**：`agent:<agent-id>:<channel>:<peer-identifier>`

---

## T

### Token（令牌）
- **定义**：用于 WebSocket 连接的认证信息
- **类型**：Gateway Token（自动生成）、API Key、OAuth Token

---

## U

## URL（统一资源定位符）
- **定义**：Web 地址的标准化路径
- **示例**：
  - 完整 URL：`https://docs.openclaw.ai/configuration`
  - 相对路径：`/configuration`
  - 参数 URL：`?param=value`

---

## V

### WebSocket
- **定义**：实时双向通信协议，用于客户端与 Gateway 通信
- **特点**：低延迟、支持流式响应、事件推送

---

## W

### Workspace（工作空间）
- **定义**：AI 智能体的文件存储和目录
- **路径**：`~/.openclaw/workspace`（默认）
- **内容**：`IDENTITY.md`、`TOOLS.md`、`SOUL.md` 等
